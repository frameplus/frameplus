# Frame Plus ERP — 회귀 테스트 체크리스트

> 배포(push) 후 라이브(`https://frameplus-erp.pages.dev`)에서 점검. 강력 새로고침(Ctrl+Shift+R) 후 시작.
> 좌상단 버전이 **v8.6.2**인지 먼저 확인(배포 반영 확인).

## 0. 배포·세션
- [ ] 강력 새로고침 후 좌상단/우하단 버전 표기 = 최신
- [ ] 개발자도구(F12) Network에서 `app.js?v=<해시>` 로 로드(해시가 직전 배포와 다름 = 캐시버스팅 동작)
- [ ] 로그인 → 로그아웃 → 재로그인 정상
- [ ] 세션 만료(또는 잘못된 세션) 시 로그인 화면으로 유도

## 1. CRUD 8개 모듈 (생성·조회·수정·삭제)
각 모듈에서 추가 → 목록 노출 → 수정 반영 → 삭제 확인.
- [ ] 프로젝트(projects) — 추가/수정/삭제, 보드 복귀
- [ ] 거래처(vendors)
- [ ] 미팅(meetings)
- [ ] 단가DB(pricedb)
- [ ] 발주(orders) — cost_type(자재/외주/일반) 구분 저장
- [ ] AS(as)
- [ ] 공지(notices)
- [ ] 세금계산서(tax) — 매출/매입 type 구분

## 2. 이번 정합성 수정 회귀 (v8.6.2 핵심)
- [ ] **계약금액 동결**: 프로젝트 견적 작성 → 계약서 자동생성 → 이후 견적 항목 수정 → **계약금액은 그대로(변동 없음)** 확인
- [ ] **정산 수금/미수금**: 프로젝트 상세 › ERP › 정산서 → 수금완료/미수금이 0이 아니라 실제 값(계약금액×수금%) 표시
- [ ] **삭제 가드**: 노무/발주/경비/세금계산서가 연결된 프로젝트 삭제 시도 → "연결된 재무기록이 있어 삭제할 수 없습니다(…건)" 안내, 삭제 안 됨
- [ ] **삭제 정상**: 재무기록 없는 프로젝트는 정상 삭제
- [ ] **메시지 템플릿 편집**: 관리자 › 메시지 템플릿 → ✏️ 편집 → 값 prefill → 저장 시 수정 반영(신규 추가가 아님)

## 3. 정산관리(글로벌 stl)
- [ ] 정산관리 메뉴 진입 시 401 없음, 레이아웃 정상(상태칩 접힘, 가로 툴바)
- [ ] 상단 배너 [dry-run 검증] → 6분류 표 표시, 0건이면 [이관할 신규 없음] 비활성
- [ ] (데이터 있을 시) [마이그레이션 실행] → confirm → 본체 반영

## 4. 금액 계산 일관성
- [ ] 견적 합계(getTotal) = 견적서 PDF 도급금액 = 계약서 도급금액 일치
- [ ] 마진율(GP) 표기가 대시보드·프로젝트·리포트에서 동일 값
- [ ] 수금률 = 수금완료 / 계약금액, 미수금 = 계약금액 − 수금완료 일치

## 5. 인쇄 / PDF
- [ ] 견적서 PDF 생성(표지·요약·집계·상세·간트·수금) 레이아웃 깨짐 없음
- [ ] 계약서 상세 인쇄/출력 정상
- [ ] 한글 폰트(Noto Sans KR) 정상 렌더(네모/깨짐 없음)
